# SecureAI — Android 모바일 앱 고도화 계획
> **문서 상태**: 제안 (Draft)
> **대상 스프린트**: 추후 백로그 편성 예정 (Sprint 16+)
> **작성일**: 2026-06-12

## 1. 개요
현재 `apps/android`는 인증서 피닝, 루팅 탐지, 오프라인 캐싱, 푸시 알림 등을 갖춘 훌륭한 "이동형 보안 대시보드(SOC)" 역할을 수행하고 있습니다.
본 문서는 B2B 엔터프라이즈 보안 솔루션이라는 특성에 맞춰 모바일 디바이스만의 고유한 기능을 활용하여 보안과 편의성을 극대화하기 위한 추가 고도화(Advanced Features) 계획을 명세합니다.

## 2. 주요 고도화 기능

### 2.1 모바일 푸시 기반 2FA 승인 (Prompt MFA)
- **배경**: 매번 6자리 숫자를 입력해야 하는 기존 TOTP 방식은 번거롭습니다.
- **기능**: 사용자가 웹(PC)에서 로그인을 시도할 때, 로그인된 안드로이드 기기로 "새로운 로그인 시도 - 승인하시겠습니까?"라는 Push 알림을 전송.
- **보안**: 앱에서 알림을 누르면 생체 인식(지문/얼굴) 후 즉시 인증 승인 처리. (Google Prompt / Duo Mobile 방식)

### 2.2 생체 인증 로그인 및 잠금 (Biometrics / FIDO2)
- **배경**: 모바일 기기에 사내 소스코드 취약점 및 주요 인프라 정보가 포함되어 있어, 기기 분실이나 타인 노출 시 위험성이 큽니다.
- **기능**: 앱 실행 또는 민감한 메뉴(예: DAST 제어 등) 진입 시 안드로이드 시스템의 `BiometricPrompt`를 호출하여 기기 소유자를 재확인.

### 2.3 긴급 차단 스위치 (Panic Button / Kill Switch)
- **배경**: 주말이나 외근 중에 심각한 제로데이 취약점 알림을 받거나, 외부 침해 사고가 의심될 때 모바일로 즉시 대처할 수 있어야 합니다.
- **기능**: 
  - 앱 내에 **[긴급 워크스페이스 락다운]** 버튼 제공.
  - 클릭 즉시 모든 활성 세션(PC 웹 포함) 강제 로그아웃.
  - 실행 중인 DAST 스캔 샌드박스 강제 중지(`kill_container` 연동).
  - 임시 조치 완료 시까지 GitHub 권한 일시 중단.

### 2.4 엔터프라이즈 모바일 관리(MDM/MAM) 호환성
- **배경**: 고객사(대기업) 보안 부서에서 사내 직원 기기에 앱을 일괄 배포할 때, 중앙 통제 설정이 필요합니다.
- **기능**:
  - Android `Managed Configurations` 지원으로, 배포 시 사내 프라이빗 백엔드 URL 동적 주입.
  - 보안 정책에 따라 앱 화면 캡처 방지(`FLAG_SECURE`) 강제 적용.
  - 필요 시 사내 VPN 앱 구동을 요구하도록 인텐트(Intent) 연동.

### 2.5 홈 화면 실시간 위젯 (App Widgets)
- **배경**: 보안 관리자는 항상 최신 보안 지표를 모니터링해야 하지만 매번 앱을 여는 것은 비효율적입니다.
- **기능**: 
  - 홈 화면에 실시간 갱신되는 대시보드 위젯 제공.
  - 주요 지표: 현재 워크스페이스 보안 점수(Security Score), 미조치 Critical 취약점 개수, 현재 실행 중인 자동화 스캔의 진행률.

### 2.6 QR 코드 스캐너를 통한 간편 온보딩 및 2FA 연동
- **배경**: 모바일 앱 초기 세팅 과정에서 긴 사내 백엔드 URL이나 2FA 시크릿 키를 입력하는 것은 오타의 위험이 있습니다.
- **기능**: 웹 대시보드 화면에 나타난 QR 코드를 안드로이드 카메라로 스캔하면 한 번에 로그인, 2FA 페어링, 백엔드 URL 설정이 완료되도록 지원.

---

## 3. 구현 우선순위 제안
1. **[Critical]** 생체 인증(BiometricPrompt) 강제화 & 화면 캡처 방지
2. **[High]** 푸시 기반 2FA 승인 (Prompt MFA)
3. **[High]** 긴급 차단 스위치 (Kill Switch)
4. **[Medium]** 홈 화면 위젯 및 QR 연동
5. **[Medium]** MDM 연동 기능
