❌ 현재 갭

VulnPanel에 체크박스 UI는 이미 존재하나, 하단 액션바가 /bulk-triage API와 연결되어 있지 않다. 단순 "패치 적용" 만 가능.

★ 신규 설계 포인트 3가지
1
이 유형 N건 스마트 배너
같은 type의 취약점이 2건 이상 선택되면 "전체 유형 선택 →" 버튼이 든 배너가 나타남. 동일 패턴을 일괄 처리하도록 유도.
2
CONFIRM / DISMISS / ACCEPT_PATCH 트리아지 액션
기존 "패치·무시" 대신 API 스펙과 1:1 매칭되는 액션 레이블. 기각(DISMISS) 사유는 append-only triage_feedback에 적재.
3
낙관적 갱신 + skip 토스트
서버 응답의 appliedVulnIds로 즉시 상태 반영. 소유X/미존재 건은 skip — UI에 "N건 처리, M건 skip" 토스트로 피드백.
API 매핑
PATCH /api/v1/vulnerabilities/bulk-triage
{ ids[], action: CONFIRM|DISMISS|ACCEPT_PATCH }
→ { requested, applied, skipped, appliedVulnIds[] }
← 왼쪽 패널이 실제 인터랙티브 시안입니다. 체크박스를 클릭해 동작을 확인하세요.
