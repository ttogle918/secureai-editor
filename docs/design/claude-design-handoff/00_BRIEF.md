# SecureAI (Pagori) — UI 리디자인 브리프 (Claude Design 핸드오프)

## 제품 한 줄
AI 기반 보안 분석 플랫폼 — **SAST(정적 탐지) + DAST(동적 증명) + 자동 패치**를 하나의 작업공간에서.
다크 OLED + 오렌지(#f97316) 액센트. (토큰: `01_DESIGN_TOKENS.md`)

## 핵심 서사 (디자인이 떠받쳐야 할 가치)
**"AI가 찾고(의심) → 샌드박스에서 실제 익스플로잇으로 증명(proven) → 자동 패치 → 검증(VERIFIED)"**
- SAST = "왜 위험한가"(의심), DAST = "정말 뚫린다"(증명). 이 **의심→증명** 격상이 제품의 차별점.

---

## 1. 페르소나 2모드 = IA 1차 축 (이미 존재)
| 모드 | 진입 | 성격 |
|---|---|---|
| **개발자** | `/editor` | 코드·취약점·패치·DAST가 한 작업공간(에디터 중심) |
| **보안담당** | 대시보드 | 통계·컴플라이언스·조직·ROI(대시보드 중심) |

레이아웃 골격(유지): `/editor` = 좌(파일트리+Monaco) / 우(취약점·패치·DAST 탭) + 상단 진행스트립 + 우하단 ChatFAB. (`02_current_components/EditorLayout.tsx`)

## 2. 데모 critical path (화면 흐름의 중심)
```
가입/온보딩(페르소나·프로젝트)
 → SAST 스캔(SSE 실시간 진행률 아코디언)
 → AST 가드(가짜 취약점 자동 폐기)
 → 취약점 상세(Monaco 포커스)
 → 트리아지(단건 ✓확인/✕기각/🩹패치채택 · ★벌크 N건 일괄)
 → DAST 증명(proven 배지 · ★다건 배치 익스플로잇)
 → 패치 자가검증(VERIFIED/FAILED 배지)
 → GitHub PR 생성(PR-only)
 → 컴플라이언스/SBOM/ROI 리포트
```

## 3. 화면별 기능 매핑 (★ = 현재 UI 공백 → 신규 설계 대상)
| 화면 | 기존 UI | ★신규 필요 |
|---|---|---|
| 취약점 패널 (`VulnPanel.tsx`) | 심각도 목록·필터 / 단건 트리아지 / DAST 결과 배지 | **★다중선택 + "이 유형 N건 선택" 배너 + 하단 벌크 액션바(확인/기각/패치채택)** |
| DAST 워크스페이스 (`DastWorkspacePage.tsx`) | 발견 엔드포인트 / Request Builder(현재 수동) | **★취약점 다건 선택 → 배치 실행 + 단일 SSE 진행 + proven 집계** |
| 패치 매니저 (`PatchManagerPage.tsx`) | PR 생성 버튼 / VERIFIED·FAILED 배지 | (있음 — 정돈·일관화만) |
| 취약점 상세 (`VulnDetailPanel.tsx`) | 코드 포커스·diff·설명 | (있음) |
| 설정 | BYOK + 모델 선택 확장 | (있음) |
| 그 외 기존 기능(컴플라이언스·SBOM·ROI·조직·모니터링·스케줄·시크릿스캔·PR자동리뷰·관리자 등) | 분산 존재 | 좌측 네비/대시보드 카드로 **그룹핑해 "도달 가능"만**(한 화면 노출 금지) |

> ★ 두 공백(**벌크 트리아지·배치 DAST**)은 백엔드·API가 이미 완료돼 있고 프론트 UI만 없다. 데모 가치가 가장 큰 곳.

## 4. (참고) 곧 추가될 데이터 — VAL-18
SAST 결과에 `attack_scenario`(공격 시나리오 1~2문장)·`tainted_parameter`(주입 파라미터)·참고링크(CWE/OWASP)·`api_endpoint`/`http_method`가 붙을 예정(ADR-017). 취약점 카드/DAST prefill 디자인 시 **이 필드들을 표시할 자리**를 미리 고려하면 좋다(지금은 없어도 됨).
