## 1. 눈에 띄는 기능 (Notable Features)

### 1.1 강점이자 핵심 차별화 포인트
1. **pgvector 임베딩 기반 보안 가이드라인 주입** (TASK-306, Sprint 3)  
   MD 파일 → `BAAI/bge-small-en-v1.5` 임베딩 → PostgreSQL IVFFlat 인덱스 → 분석 시 스택별 자동 삽입 → Prompt Caching 활성화.  
   단순 룰셋이 아닌 "컨텍스트 인식" 분석이라는 차별점이 가장 잘 드러나는 기능.

2. **파일 SHA-256 해시 캐싱 (ADR-007)**  
   동일 파일 재분석 방지 → API 비용·분석 시간 동시 절감. 캐시 키 `secureai:sast:{sha256}:{language}` 설계가 세분화되어 있음.

3. **GDPR 소프트→하드 삭제 이중 구조 (Sprint 9, TASK-907)**  
   `deleted_at` 소프트 삭제 → 30일 경과 후 배치 50건 단위 연쇄 삭제 → 감사 로그 선행 기록 → ApplicationEvent 크로스도메인 삭제. 단순 DELETE가 아닌 법적 요건과 아키텍처 원칙을 동시에 충족.

4. **보안 문서 자동 생성 3종 (TASK-MISC-002)**  
   Thymeleaf + OpenHTMLtoPDF. CISO 보고서·행안부 체크리스트·ISMS-P 증적을 SAST 결과에서 자동 변환. 기업 고객 페인포인트를 정확히 공략하는 기능.

5. **MCP thin-wrapper DAST 아키텍처 (Sprint 9, TASK-905)**  
   AI Engine → MCP → Backend HTTP → Docker SDK 경로. Option B(thin wrapper) 채택으로 `DastExecutionService` 격리 정책 유지하면서 MCP 호환성 확보. 아키텍처 결정 과정이 문서에 명확히 남아 있음.

6. **VSCode Extension Diagnostic API 연동 (TASK-902)**  
   Problems 탭 직결은 "IDE 네이티브 경험"을 제공하는 가장 파급력 있는 UX. SecretStorage JWT 저장으로 보안도 고려.

## 3. 보완점 (Areas for Improvement)

- 가이드라인이 오래될수록 분석 정확도 저하 가능성.
- **조치**: `NvdSyncJob`과 유사하게 `GuidelineSyncJob` 스케줄러 추가(주 1회 MD 파일 변경 감지 + 임베딩 재생성).

### 최종 총평

> SecureAI Editor의 설계 문서군은 **"팀이 의사결정의 이유를 기억하는 시스템"** 을 구축하는 데 성공했다. ADR·스프린트 문서·Reviewer 게이트 기록이 삼위일체로 작동하여 구현 과정의 트레이드오프가 손실 없이 보존되어 있다.
>
> 개선이 필요한 영역은 기술 수준보다 **"문서 위생(hygiene)"** 에 집중된다. 버전 파일 중복, ADR 갱신 지연, 이월 태스크 누적이 앞으로 팀 규모가 커질 때 온보딩 비용과 혼란을 가져올 수 있다. Sprint 10 착수 전에 위 수정 항목을 처리하고 문서를 현행화하는 것을 권장한다.
>
> 이후 "구현 코드와 설계 비교 재평가"에서는 특히 ① `17_ARCHITECTURE_CURRENT_V2.md`의 다이어그램과 실제 Docker Compose 구성 일치 여부, ② ADR-004 스케줄 목록과 실제 `@Scheduled` Job 전수 대조, ③ MCP SQL 예외 처리 코드 패턴 현황을 중점적으로 확인할 것을 권장한다.
