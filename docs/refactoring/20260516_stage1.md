## 4. AI 서버 통신 지점 (핵심)

### 4.1 Backend → AI Engine (Spring RestClient)

| 파일 | 클라이언트 | 엔드포인트 | 인증 |
|------|-----------|-----------|------|
| `DefaultAiAgentClient.java` | `RestClient` (JDK HttpClient 기반) | `POST /agent/analyze` | `X-Internal-Key` |
| `DefaultAiAgentClient.java` | `RestClient` | `POST /agent/resume/{sessionId}` | `X-Internal-Key` |
| `DefaultAiAgentClient.java` | `RestClient` | `POST /agent/cancel/{sessionId}` | `X-Internal-Key` |
| `DefaultAiAgentClient.java` | `RestClient` | `POST /agent/translate` | `X-Internal-Key` |
| `ChatService.java` | `RestClient` | `POST /agent/chat` | `X-Internal-Key` |
| `CommitSecretService.java` | `HttpClient` (java.net) | `POST /agent/secret_scan` | `X-Internal-Key` |

> `DefaultAiAgentClient`는 자체 Circuit Breaker를 내장 (실패 3회 → OPEN, 30초 후 HALF-OPEN 시도).

## 8. 발견된 혼용 패턴 및 개선 포인트

| 위치 | 현황 | 비고 |
|------|------|------|
| `AppConfig.restTemplate()` Bean | 레거시 `RestTemplate` 등록 | `GitHubOAuthService`만 사용 |
| `GitHubOAuthService` | `RestTemplate` 사용 | 나머지 GitHub 클라이언트는 `RestClient` |
| `CommitSecretService` | `java.net.HttpClient` 직접 생성 | `RestClient`로 통일 가능 |
| `DomainVerificationService` | `java.net.HttpClient` 직접 생성 | `RestClient`로 통일 가능 |
| `DefaultAiAgentClient` Circuit Breaker | 수동 구현 | Resilience4j 도입 검토 여지 |
| AI Engine `backend_api_client.py` | `httpx.AsyncClient` per-call 생성 | 연결 풀 재사용 최적화 여지 |
