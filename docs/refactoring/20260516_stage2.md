## 4. AI 서버 통신 로직 분리 여부

> 기준: 전용 Client 클래스를 통해서만 AI Engine에 접근해야 함 (`AiAgentClient` 인터페이스)

| 파일 | 문제 | 심각도 | 개선 방향 |
|------|------|--------|---------|
| `domain/dast/controller/DastController.java` L56-61, L112-133 | `@PostConstruct`에서 `RestClient` 생성 후 컨트롤러 내부에서 `/agent/dast/start` 직접 호출. Circuit Breaker 없음 | **상** | `AiAgentClient` 인터페이스에 `startDast(DastStartPayload)` 추가 → `DefaultAiAgentClient`가 구현. Controller는 Service → AiAgentClient 경로만 사용 |
| `domain/analysis/service/ChatService.java` L44-68 | 생성자에서 `RestClient` 직접 빌드 후 `/agent/chat` 호출. `AiAgentClient` 인터페이스를 경유하지 않음. Circuit Breaker 없음 | **상** | `AiAgentClient`에 `streamChat(sessionId, message, history): InputStream` 추가하거나 전용 `AiChatClient` 인터페이스 분리. ChatService는 클라이언트 의존 |
| `domain/analysis/service/CommitSecretService.java` L45-57 | `@Value` setter로 `RestClient` 지연 생성 후 `/agent/scan-commits` 직접 호출. 인터페이스 없음, Circuit Breaker 없음. `X-Internal-Key`를 헤더에 per-request로 추가 (설정 중복) | **중** | `AiAgentClient`에 `startCommitScan(...)` 메서드 추가. `internalApiKey`는 `DefaultAiAgentClient` 생성자에서만 설정 (헤더 중복 제거) |

**정상 패턴 (참조)**

| 파일 | 구현 방식 |
|------|---------|
| `DefaultAiAgentClient.java` | `AiAgentClient` 인터페이스 구현, Circuit Breaker 내장 (3회 실패 → OPEN, 30초 HALF-OPEN), X-Internal-Key 헤더 중앙 관리 |
