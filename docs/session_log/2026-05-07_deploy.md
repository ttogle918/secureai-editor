## 3. 배포 구조 상 주요 사항

### docker-compose.yml과 Railway의 관계
Railway는 `docker-compose.yml`을 읽지 않는다.  
`docker-compose.yml`은 로컬 개발 전용. Railway에서는 각 서비스를 개별로 추가.

### MCP Server 처리
AI Engine이 MCP Server를 subprocess로 spawn하므로 **상시 실행 컨테이너** 필수.  
서버리스(Lambda, Vercel Functions)에서는 실행 불가.  
Railway 서비스는 컨테이너가 항상 떠 있으므로 문제 없음.  
AI Engine Dockerfile에 Node.js 설치 포함 → MCP Server spawn 정상 동작.

### 내부 통신 주소
Railway private network에서 서비스 간 통신:
- `AI_AGENT_URL` = `http://ai_engine.railway.internal:8000`
- `BACKEND_INTERNAL_URL` = `http://backend.railway.internal:8080`

## 4. 배포 전 필수 보안 조치

| 항목 | 기본값 상태 | 조치 |
|---|---|---|
| `SECUREAI_ENCRYPTION_KEY` | `000...000` (64자리 0) | `openssl rand -hex 32` 로 교체 필수 |
| `JWT_SECRET` | `application.yaml`에 하드코딩 노출 | `openssl rand -base64 64` 로 교체 필수 |
| `MAIL_PASSWORD` | 미설정 | Gmail 앱 비밀번호 생성 후 입력 |
| `INTERNAL_API_KEY` | `changeme_internal_secret` | `openssl rand -hex 32` 로 교체 |

> `SECUREAI_ENCRYPTION_KEY`가 0으로만 채워지면 저장된 모든 API 키가 동일 키로 암호화됨.  
> DB에 데이터 없는 지금 교체하면 마이그레이션 불필요.
