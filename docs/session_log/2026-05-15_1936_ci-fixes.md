### CI 실패 2 — Mockito `UnnecessaryStubbingException` (SbomServiceSaveTest, CommitSecretServiceTest)

`@ExtendWith(MockitoExtension.class)`는 strict stubbing을 적용하므로 `@BeforeEach`에서 세팅한 stub이 특정 테스트에서 전혀 호출되지 않으면 `UnnecessaryStubbingException`을 던짐.

- `SbomServiceSaveTest`: `sessionRepository.findById(sessionId)` / `projectRepository.findById(projectId)` 두 stub이 "unknownSession" 케이스(TC-4)에서 미사용 → `lenient().when()`으로 전환
- `CommitSecretServiceTest`: RestClient 체이닝 stub 7개 (`.post()`, `.uri()`, `.contentType()`, `.header()`, `.body()`, `.retrieve()`, `.toBodilessEntity()`)가 SESSION_NOT_FOUND·ACCESS_DENIED를 던지는 테스트에서 미사용 → `lenient().when()`으로 전환

### CI 실패 3 — `CommitSecretServiceTest` TC-3 NPE

UnnecessaryStubbingException을 lenient() 로 해소한 후 드러난 잠재 버그.

**원인**: `header(anyString(), anyString())`이 `header(String name, String... values)` varargs 메서드와 매칭에 실패. Mockito가 반환한 `null`에 대해 `.body(body)` 호출 → NPE.

**결정**: 개별 체인 stub 7개를 제거하고 `RETURNS_DEEP_STUBS`로 대체.

```java
// Before: 수동으로 체인 각 단계 stub (varargs 불일치 위험)
RestClient.RequestBodyUriSpec uriSpec = mock(RestClient.RequestBodyUriSpec.class);
...
lenient().when(bodySpec.header(anyString(), anyString())).thenReturn(bodySpec);

// After: RETURNS_DEEP_STUBS — 체인 전체를 null 없이 자동 목킹
RestClient mockRestClient = mock(RestClient.class, Mockito.RETURNS_DEEP_STUBS);
```

TC-6(AI Engine 호출 실패 케이스)은 테스트 내부에서 `agentRestClient`를 별도 목으로 교체해 `post().thenThrow()` 하므로 영향 없음.

### CI 실패 4 — ESLint `no-explicit-any` 미정의 규칙 오류

`AppSidebar.tsx:40`의 `// eslint-disable-next-line @typescript-eslint/no-explicit-any` 주석이 문제. 프로젝트 ESLint 설정(`"extends": ["next/core-web-vitals"]`)에 `@typescript-eslint/eslint-plugin`이 로드되지 않아 해당 규칙을 알 수 없다는 에러로 처리됨.

**결정**: `any` 자체를 제거하는 방향으로 수정.

```typescript
// Before
// eslint-disable-next-line @typescript-eslint/no-explicit-any
type Branch = Record<string, any>;

// After — 재귀 타입으로 any 불필요
type Branch = { [key: string]: number | Branch };
```

`Branch`가 `number`(파일 랭크) 또는 재귀 `Branch`(디렉터리)만 가질 수 있는 구조이므로 타입 표현이 더 정확해짐.
