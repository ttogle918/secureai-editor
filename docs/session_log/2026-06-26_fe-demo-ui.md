## 2. 의논 내용 & 결정 맥락

### 2.1 design-sync 시도 → 직접 구현 전환
- **상황**: Claude Design에서 산출물(IA Audit·Dashboard·Editor 리디자인 .dc.html + _ds 번들)을 받음 → `design-sync` 스킬로 컨버전 시도
- **장애**: 이 레포는 Next.js 앱이지 Design System 라이브러리가 아니라서, design-sync의 컨버터가 `node_modules/<pkg>` self-install을 찾지 못함. synth-entry도 동작 불가
- **결정**: design-sync 접고 **시안을 직접 코드 구현**으로 전환 (사용자 "둘 다 할 수 있으면 구현 우선" 지시)
- **이유**: 사용자가 이미 상세한 디자인 .html 결과를 만들었기에, 구현이 더 빠르고 실제 제품 검증도 가능

### 2.2 벌크 트리아지 오배치 발견 & 재이식
- **발견**: Dev 에이전트가 구현한 "벌크 트리아지" 기능이 `VulnPanel.tsx`에 있었으나, **이 컴포넌트는 앱에서 렌더되지 않는 레거시**(grep·git 히스토리상 참조 0)
- **실제 구조**: editor 오른쪽 패널은 `RightPanel` → `VulnDetailPanel` (현재 활동 컴포넌트)
- **근본원인**: session_log §4, feature_inventory, Claude Design 시안이 모두 "VulnPanel"을 언급했으나, 실코드는 VulnDetailPanel 사용 → **명명 불일치**
- **해결**: 
  - 벌크 트리아지 로직(멀티셀렉트, store 업데이트)을 VulnDetailPanel로 재이식
  - unused VulnPanel은 피처 전 상태로 revert
  - **사용자 요청**: 행 클릭 = 멀티셀렉트 토글, 상세보기 = chevron 분리 (UX 개선)
- **영향**: 실제로 동작하는 패널에 기능이 이제 연결됨

### 2.3 단계별 구현 합의 (B → C → D)
- **현황**: Claude Design이 앱 IA 전체 재설계라 한 번에 모두 못 밀 수 있음
- **합의 경로**: **B(사이드바 재편)** → **C(대시보드/헤더)** → **D(에디터 스플릿)**로 3단계 구분
- **커밋 전략**: 각 단계 완료 후 별도 커밋 (진행 상황 추적 용이)
- **파일트리 폴더접기**: 이미 코드상 동작 확인 → 추가 작업 불필요

### 2.4 정직성 원칙: 가짜 데이터 금지
- **IA 묻힌 기능 정리**:
  - **라우팅 연결** (FE 페이지 있음): commit-scan, github-scan, sbom, compliance, admin, team
  - **비활성 처리** (FE 페이지 없음): PR자동리뷰, 예약스캔, 도메인모니터링 → "준비 중" 비활성
- **대시보드 데이터**:
  - **크레딧**: `/users/me` 실데이터 연동 (useCredits 훅)
  - **컴플라이언스/모니터링**: nav에는 표시하되 "준비 중" 상태 (가짜 % 진행률 금지)
- **github-scan 회귀**: B에서 제거됐다가 실제 페이지 있음 확인 후 복구

### 2.5 design-sync 검증 & 데모 라우트
